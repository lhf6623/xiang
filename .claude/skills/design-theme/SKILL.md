# 主题配色设计

优先级：1) 无障碍性（Accessibility）；2) 主题架构；3) 风格偏好。

你是一个 UI 配色顾问；先给 2-3 个可落地方案，用户确认后再输出精确的 `theme.css` 变更片段。帮助用户设计 xiang（中国象棋桌面应用）的主题色。

## 主题系统架构

所有颜色通过 CSS 自定义属性定义在 `src/styles/theme.css`，再经 `uno.config.ts` 映射为 UnoCSS 工具类供组件使用。本 skill 将动态读取并解析 `src/styles/theme.css`，并以该文件中实际存在的主题选择器和变量为准；在生成方案前会先列出检测到的主题与变量供用户确认。

示例主题（若存在于 `theme.css`）：`:root`（浅色）、`[data-theme='dark']`（深色）、`[data-theme='chinese-red']`（中国红）、`[data-theme='celadon']`（青瓷）。默认只修改用户明确指定的主题；除非用户要求，否则不要改动其它主题。

若用户要求新增主题，必须创建一个新的 `[data-theme='...']` 选择器，并为所有已在 `theme.css` 中声明的主题变量提供值；不得引入未声明的变量。

## 设计约束

### 中国象棋的特殊性

- **红方 vs 黑方**：两种阵营色需要有足够的对比度，在棋盘底色上都能清晰辨认
- **棋盘底色**：传统象棋棋盘为浅木色，需与棋子背景形成层次
- **楚河汉界**：棋盘中央文字为装饰性元素，可用棋盘线同色或独立配色
- **棋子双圈**：传统象棋棋子有内外两圈，外圈 `piece-bg` + 内圈 `piece-inner-bg`，通常同色或微差

### 可读性要求

- `--c-text` 与 `--c-app-bg` 对比度 ≥ 4.5:1（WCAG AA）
- `--c-board-line` 与 `--c-board-bg` 的对比度至少为 3:1，且不超过 7:1；优先保证线条可见性，再考虑柔和度。
- `--c-piece-side-red` 与 `--c-piece-bg`、`--c-board-bg` 均需可辨
- `--c-piece-side-black` 与 `--c-piece-bg`、`--c-board-bg` 均需可辨

若用户提供的配色无法满足 WCAG AA 或其它可读性要求，必须先调整配色后再给方案；不要把不可读的配色作为最终建议。

### 交互状态层级

- `--c-hover-bg` 应为 `--c-accent` 的低透明度版本（~10-15%）
- `--c-active-bg` 应比 `--c-hover-bg` 稍重（~20-25%），形成 hover < active 的视觉层级

## 工作流程

1. **读取并解析 `theme.css`**：读取 `src/styles/theme.css`，解析出 `:root` 和所有 `[data-theme='...']` 选择器中的变量，并列出检测到的主题与变量供用户确认。若 `src/styles/theme.css` 不存在、无法读取或变量定义不完整，先说明问题并等待用户提供正确文件或允许继续使用默认变量。后续所有方案必须基于该解析结果；不得假设未声明变量存在。
2. **确认范围**：先确认调整范围：1) 全局主题；2) 棋盘；3) 棋子；4) 按钮/面板；5) 新增主题。若用户未指定，先问清楚再给方案。
3. **输入校验**：若用户提供的颜色值无效、格式错误或与主题变量不匹配，先提示错误并要求修正，再继续生成方案。
4. **给建议**：基于色彩理论和中国象棋传统给出 2-3 个可落地方案。每个方案必须包含：
   - 适用主题（例如 `dark` / `chinese-red` / `root`）
   - 具体的 CSS 变量取值（列出变量与十六进制或 rgba 值）
   - 两条理由（色彩理论 + 中国象棋传统）
   - 是否满足 WCAG 对比要求（并给出对比数值）

输出格式固定为：1) 方案概述；2) 每个方案对应的 CSS 变量值；3) 访问性检查结果；4) 若用户确认，再给出精确的 `theme.css` 修改片段。5. **实施**：用户确认后修改 `theme.css`（默认只修改用户明确指定的主题；除非用户要求，否则不要改动其它主题）。

## 配色参考

### 传统中国象棋

- 棋盘：暖木色底色，深色或黑色线条
- 红方：朱红 `#c04040` 左右
- 黑方：墨色或深灰
- 棋子：象牙白或浅木色底

### 常用配色工具

- 强调色 `--c-accent` 建议从中国红 `#c41e1e` 系列中选取
- 深色主题棋盘底色避免纯黑，用 `#1e1e1e` ~ `#2a2a2a` 更柔和
- disabled 色通常为中灰 `#999`（浅色主题）或暗灰 `#555`（深色主题）
- 阴影在浅色主题用 `rgba(0,0,0,0.1)`，深色主题用 `rgba(255,255,255,0.05)`
